# melsec_mc_mock — モックサーバ仕様

このドキュメントは `melsec_mc_mock` クレートの設計と実装上の振る舞い（モック PLC）を示します。

- 入出力契約
  - 受信側（PLC）はライブラリのクライアントから来たリクエストを受け取り、論理的なペイロード（デバイスの読み書きデータ）だけを返します。
  - 送受信のフレーム化（サブヘッダ／アクセスルート／エンドコードの付与）は transport 層が担います。

注意: リポジトリ全体で用語を統一するため、従来の「ハンドラ」表記は `受信側（PLC）` に置き換えています。これはユーザーが混乱せず、ライブラリの目的（PLC との通信）が明確になるためです。
